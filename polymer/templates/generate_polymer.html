Functional Overview

Polymer Design Interface: Users can design polymers by selecting the desired polymerization method from four supported types: homopolymer, alternating copolymer, block copolymer, and random copolymer. For each component, users are required to provide the monomer name, corresponding SMILES notation, and the number of repeating units. This information forms the foundation for polymer construction and downstream modeling.

1. Predict Polymer: Property prediction is performed based on the methodology proposed in the paper "OpenPoly: A Literature-Derived Polymer Database Empowering Benchmarking and Multi-Property Prediction". For homopolymers, the entire SMILES is evaluated directly. For copolymers, a component-contribution approach is adopted: the model first predicts properties for each individual monomer unit, then computes a weighted average according to the number of repeating units of each component, thereby estimating the final properties of the copolymer.

2. Generate Polymer: Polymer chains with arbitrary degrees of polymerization are constructed using atomic-level editing techniques. The structural files are generated in multiple formats, including .pdb, .xyz, and .mol2, based on adjacency and node matrices. Additionally, the topology of the polymer is defined using sobtop, and atomic charges are assigned using the RESP scheme. This ensures compatibility with molecular dynamics simulations and facilitates high-fidelity polymer modeling.
